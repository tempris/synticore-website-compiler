# cure-gulp-file-include

In-house Synticore fork of `gulp-file-include`.

## Key differences from upstream

- Removes `new Function(...)` + `with(...)` runtime execution for `@@if` and `@@for`.
- Uses custom directive parsing (balanced scanning) instead of brittle regex-only argument parsing.
- Detects include recursion across full include stack, not just direct self-include.
- `@@loop` supports a structured 3rd options object argument:
  - `@@loop('template.html', 'data.json', { "loop": { "item_max": 6 } })`
  - `@@loop('template.html', 'data.json', { "loop": { "filter": { "category": "news" } } })`
  - `@@loop('template.html', 'data.json', { "loop": { "filter": "item.category === context.category_filter" } })`
  - `@@loop('template.html', 'data.json', { "loop": { ... }, "context": { "style_column": "col-4" } })`
- `@@include` supports collection-selection data resolution:
  - `@@include('article-footer.html', { "from": "articles.json", "select": "item.link === context.article_link", "neighbors": true, "context": { "article_link": "/post" } })`
  - Resolves `current`, `previous`, `next`, `_index`, and `_length` for the included partial
- `@@include_once` dedupes by normalized resolved file path.
- Adds richer error diagnostics with file/line/column + include stack.
- Uses deterministic indentation when `indent: true`.
- Uses strict filter handling and explicit unknown-filter errors.

## Compatibility notes

- Kept synchronous file I/O by design.
- Standard syntax remains supported:
  - `@@include(...)`, `@@include_once(...)`, `@@loop(...)`, `@@if (...) { ... }`, `@@for (...) { ... }`
